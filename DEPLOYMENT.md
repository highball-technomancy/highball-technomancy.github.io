# Deployment Guide - Highball Technomancy Website

## Quick Start (First Time Setup)

### 1. Install Dependencies

Open PowerShell in the project directory and run:

```powershell
npm install
```

This will install Astro, Tailwind, and all dependencies.

### 2. Test Locally

Run the development server:

```powershell
npm run dev
```

Visit `http://localhost:4321` in your browser. You should see your site with:
- âœ… Animated logo (highball glass with neon effects)
- âœ… Particle background (floating binary code)
- âœ… Smooth scrolling navigation
- âœ… All sections (About, Products, Technology, Contact)
- âœ… Neon glow effects on hover

### 3. Build for Production

Generate the static site:

```powershell
npm run build
```

This creates optimized files in the `/docs` directory (which GitHub Pages serves).

### 4. Commit and Push

```powershell
git add .
git commit -m "Initial Highball Technomancy website"
git push origin main
```

### 5. Configure GitHub Pages

1. Go to your repo: `https://github.com/highball-technomancy/highball-technomancy.github.io`
2. Click **Settings** â†’ **Pages** (left sidebar)
3. Under **Source**, select:
   - Branch: `main`
   - Folder: `/docs`
4. Click **Save**

GitHub will automatically deploy. Wait 1-2 minutes, then visit: `https://technomancyai.com`

## Making Updates

### Update Content

1. **Edit the source files** in `/src`:
   - `src/pages/index.astro` - Main page content
   - `src/components/*.astro` - Individual components
   - `src/styles/global.css` - Global styles

2. **Test your changes:**
   ```powershell
   npm run dev
   ```

3. **Build and deploy:**
   ```powershell
   npm run build
   git add docs/
   git commit -m "Update website content"
   git push origin main
   ```

### Update Logo

**Option A: Replace SVG component**
- Edit `src/components/Logo.astro`
- Change the SVG paths, colors, or structure

**Option B: Use an image file**
1. Save your logo as `logo.png` or `logo.svg` in `/public`
2. In `src/pages/index.astro`, replace:
   ```astro
   <Logo size="300" class="animate-float" />
   ```
   with:
   ```astro
   <img src="/logo.png" alt="Logo" class="w-[300px] animate-float" />
   ```

### Update Colors

All colors are in `tailwind.config.mjs`. To change:

```javascript
colors: {
  'neon-green': '#00c853',  // Change this hex code
  'neon-purple': '#a855f7', // Change this hex code
  // etc.
}
```

Then rebuild and deploy.

## Troubleshooting

### Port 4321 Already in Use

```powershell
# Kill the process
npx kill-port 4321

# Or use a different port
npm run dev -- --port 3000
```

### Build Fails

```powershell
# Clear Astro cache
Remove-Item -Recurse -Force .astro

# Reinstall dependencies
Remove-Item -Recurse -Force node_modules
npm install

# Try building again
npm run build
```

### GitHub Pages Not Updating

1. **Check `/docs` was committed:**
   ```powershell
   git status
   ```
   If `/docs` isn't staged, add it:
   ```powershell
   git add docs/
   git commit -m "Update built site"
   git push origin main
   ```

2. **Check CNAME file exists:**
   ```powershell
   Get-Content docs/CNAME
   ```
   Should show: `technomancyai.com`

3. **Check GitHub Pages settings:**
   - Repo â†’ Settings â†’ Pages
   - Source: `main` branch, `/docs` folder
   - Custom domain: `technomancyai.com`

4. **Wait 2-3 minutes** for GitHub to rebuild. Check the Actions tab for deployment status.

### Custom Domain Not Working

1. **DNS Settings** (at your domain registrar - Spectrum):
   - Add **A records** pointing to GitHub's IPs:
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
   - Add **CNAME record**:
     ```
     www  â†’  highball-technomancy.github.io
     ```

2. **Wait for DNS propagation** (can take 24-48 hours)

3. **Verify DNS:**
   ```powershell
   nslookup technomancyai.com
   ```

### Animations Not Working

Make sure you're viewing over HTTPS (`https://technomancyai.com`), not HTTP.

Some browser extensions (ad blockers) can interfere with animations.

### Favicon Not Showing

Clear browser cache:
- Chrome: `Ctrl + Shift + Delete`
- Hard refresh: `Ctrl + Shift + R`

Favicon changes can take time to propagate.

## Development Workflow

### Recommended Flow

1. **Start dev server:** `npm run dev`
2. **Make changes** in `/src` files
3. **Browser auto-refreshes** (hot reload)
4. **Test thoroughly** on `localhost:4321`
5. **Build:** `npm run build`
6. **Preview production build:** `npm run preview`
7. **Commit and push:** Git commands
8. **Wait for GitHub Pages** to rebuild

### File Structure Reminders

```
/src          â†’ Source files (you edit these)
/docs         â†’ Built site (generated by `npm run build`)
/public       â†’ Static assets (copied as-is to /docs)
/node_modules â†’ Dependencies (don't commit)
```

**Never edit files in `/docs` directly** - they get overwritten on every build.

## Performance Tips

### Optimize Images

If you add images to `/public`:
1. Compress them first: [TinyPNG](https://tinypng.com)
2. Use WebP format when possible
3. Provide width/height attributes

1. Visit your live site: `https://technomancyai.com`

## Backup Strategy

### Before Major Changes

```powershell
# Create a backup branch
git checkout -b backup-$(Get-Date -Format "yyyy-MM-dd")
git push origin backup-$(Get-Date -Format "yyyy-MM-dd")

# Return to main
git checkout main
```

### Rollback if Needed

```powershell
# See commit history
git log --oneline

# Rollback to a specific commit
git reset --hard <commit-hash>
git push origin main --force
```
